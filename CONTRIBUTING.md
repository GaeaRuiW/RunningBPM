# 贡献指南

感谢您对 RunningBPM 项目的关注！我们欢迎所有形式的贡献。

## 如何贡献

### 报告问题

如果您发现了 bug 或有功能建议，请：

1. 检查 [Issues](https://github.com/yourusername/RunningBPM/issues) 中是否已有相关问题
2. 如果没有，请创建新的 Issue，详细描述问题或建议

### 提交代码

1. Fork 本仓库
2. 创建您的特性分支 (`git checkout -b feature/AmazingFeature`)
3. 提交您的更改 (`git commit -m 'Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 开启一个 Pull Request

### 代码规范

- 遵循 PEP 8 (Python) 和 ESLint (TypeScript/JavaScript) 规范
- 添加适当的注释和文档字符串
- 确保所有测试通过
- 更新相关文档

### 开发环境设置

请参考 README.md 中的快速开始部分。

## 行为准则

- 尊重所有贡献者
- 接受建设性的批评
- 专注于对项目最有利的事情
- 对其他社区成员表示同理心
